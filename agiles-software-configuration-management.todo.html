Agiles Software Configuration Management

Ich habe mich in den letzten Wochen etwas intensiver mit der Erstellung und Konfiguration von Software beschäftigt. Mittlerweile gehört es ja schon zum guten Ton ín der Software-Entwicklung, zumindest ein Continuous Build zu haben - besser noch komplett die Methoden der Continuous Integration anzuwenden. Es gibt ja auch inziwschen eine Reihe von Tools, die ein solches Vorgehen gut unterstützen - TFS, CruiseControl.NET, NAnt oder TeamCity - um nur einige zu nennen.

Entscheidend bei der Anwendung von Continuous Integration ist aber nicht nur das Toolset, sondern in großem Maße auch die "Theorie des Softwarebau's" - dem Software Configuration Management. Ein wenig komisch ist diese Beziehung zwischen dem agilen Continuous Integration und dem formalistischen Software Configuration Management schon. Bei näherer Betrachtung jedoch nehmen die Gemeinsamkeiten gegenüber den Unterschieden deutlich überhand.

Die wenigsten Software-Entwickler können sich unter dem Begriff Software Configuration Management (SCM) etwas konkretes vorstellen. Meistens wird SCM auf ein Versionskontrollsystem wie TFS, SVN, GIT oder Perforce reduziert. SCM ist jedoch weitaus mehr. In dem Artikel "Grundlagen des agilen Software Configuration Management" habe ich die Grundlagen des SCM zusammengefasst und den theoretischen Stoff ein wenig aufgearbeitet. Hoffentlich hilft es dem einen oder anderen, diese theoretischen Konzepte auf den praktischen Einsatz abzubilden.

written on 07 February 2009
by Ilker Cetinkaya.
