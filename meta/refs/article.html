A reference title is a reference title no matter how long that title is.

This is a reference article (or blog post) page. It contains text as well as other media. For reference purposes, this page will contain a variety of text format types and variations, in order to have a good overview on how the real article pages might look like when they are using specific text formatting classes.

Let's just start with two separate paragraphs in order to see how they are visually separated - just by the paragraph spacing itself.

Common text formatting

This reference text already used some of the inline formatting styles. Apart from the common styles like emphasis (italic) and strong (bold), there's a special inline format for monospace or typewriter or code fragments.

Naturally, inline text may contain References to other pages as well. Note that most probably there's no visual difference between visited and active links. However, the visualization might change when you hover over a link.

Although it's widely used in the internet, common emoticons like :-) smilies ;-) and others :P most probably are displayed as is.

Interestingly, citations and quotes are being used quite often.

Listings

Sometimes it's useful to list things up. Like an enumeration of things to consider when you want to cook a nice dish. An unordered listing might look like this:

  • 4 Eggs
  • 200g Half-bitter chocolate
  • 1 liter of milk
  • some strawberries
  • actually someone who understands cooking.

A second form of listing is the ordered list:

  1. A good set of things to list
  2. A good sense of meta-thinking
  3. The requirement to list them orderedly
  4. Actually listing them in the document

Some interesting - more or less exotic - constellations with listings:

  • A listed item which spans
    over two or more
    lines
  • A listed item containing some inline formatting including code
  • A listed item which contains a

    quote within

Granted, the above examples won't occur that often.

Images

A very interesting thing actually is the inclusion of images. Embedding images to articles is a good thing. It emphasizes and illustrates the topic on which the whole content is all about.

By convention, two sizes of images are supported to be embedded into an article page. The above image is the larger format. It's single constraint it to have a maximum width of 640px.

The second, small sized image format is scales to 150px in both width and height. This "smaller" (thumbnail) version of images are well suited to be placed floating right or left beneath the actual content.

One of the noteworthy features when it comes to embedding images is the "illustration" (or figure) mode.

With the known extended annotation syntax in Markdown, the embedded image can be decorated with a figure statement (descriptive text). Even more, the class annotations can be utilized to align the embedded thumbnail on the left or right hand side.
