SQLPlus on Linux

Lately, I had to do some little hacking with Oracle databases again. I'm kind of a fan of the Oracle database philosophy and overall system layout. Despite Forms and Java, of course. But that's a story of its own, actually. Other RDBMS are surely great as well.

As it is with everything shining, there's shadow as well. One of the things I'm really sort of irritated every time again and again is the weak onboard toolset. One of the core tools every Oracle developer or DBA is required to deal with certainly is SQLPlus. I like using SQLPlus, since it gives me a quick REPL plus some additional script execution features.

Having this extraordinary importance of SQLPlus in mind, it's sort of ridiculous that SQLPlus does not even support the most simple command line features such as command history, cursor movement or history search. More precisely, SQLPlus doesn't support this for Linux boxes actually. This lack of usability on the CLI sometimes can be a real pain. The reason for this uncomfy behavior in fact is quite simple: SQLPlus wasn't developed with a decent (read: human friendly) character input implementation.

How lucky we are to have Linux as a platform! Why? Well, because we surely have a program to fix this issue. And indeed there is a magic tool which enables us to use SQLPlus with all the comfort of a standard Bash shell: rlwrap. Just install it, type rlwrap sqlplus, and suddenly SQLPlus happens to be user-friendly! Obviously, rlwrap can be used as a wrapper for any CLI program to enrich it with readline features.

written on 19 July 2013
