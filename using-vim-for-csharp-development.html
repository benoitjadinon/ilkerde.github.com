Using VIM for C# Development

It's been a few years now since I came back to VIM after a decade of absence. I remember it was around the almighty Y2K-Bug being my most active time with VIM. Then, Visual Studio reigned my desktop for almost a decade. In retrospective, I have to say: By that time I was just using 1% of the true power of VIM - yet I felt somehow productive with it.

Needless to say that today I'm feeling more productive than ever with VIM. Of course, today I'm using a lot more than 1% of VIMs super powers. Maybe around 2% or something like that.

I'm still an avid user of Visual Studio (and ReSharper), because it simply is an amazing IDE. It helps me alot in my daily work as well as for large-scale .NET development. Even more, I'm pretty much sold to ReSharper. Since 2010, I really love to use Visual Studio with ReSharper when it comes to C# development. From my gut feeling, I think I'm using around 30% to 40% of VS/ReSharpers capabilities.

VIM and C#

With all that love for C# and VS/ReSharper, there's still the notion and need for me to get back to my beloved VIM even for C# development. I'm happily using Linux now for years and of course do develop stuff on Linux as well. Yes, I do use MonoDevelop from time to time as well, but I always come back to the point where I think to myself: "I don't feel comfortable here, VIM is nearby, so why don't use it?"

Since 2011 I'm using VIM for (small) C# projects. "Small" in this context is something between five and ten assemblies (csproj projects). I started with doing Katas in VIM, just to know how it feels to have no compiler at F5 or an external unit test runner. Then, I gradually started to edit projects. I realized that grep and sed will stand beside me for my project management needs.

But still, I somehow couldn't reach that level of comfort that I would actually feel confident to entirely use VIM for small C# projects. Until beginning of 2013. Then, a new era started for me and my C# development desires: I discovered Omnisharp!

Omnisharp!

Omnisharp is such an incredibly great thing. It gives me what I thought of could be rarely ever possible: Intellisense! Even more, it delivers me Type Lookup and Symbol Navigation. Yes, all you spoilt Visual Studio "power users", I'm excited about this! And just to let you know: there's even Refactorings available!

The real value of Omnisharp plugin truly is that finally I can combine the superior editing of VIM with the essential symbol navigation, lookups and intellisense. I already did a few (little) projects with VIM and Omnisharp and have to conclude: Now I'm (almost) as fast and productive with VIM as I am with VS/ReSharper. For me, this means that I my little dream of pure C# development with VIM becomes a real option.

As of today, I already do all of my personal code review and code contributions with VIM. Even more, I comfortably took part in 2 little projects with VIM as well. I'm not sure if I want to use VIM/Omnisharp for large projects as of today, but I'll definitely continue to use it as my first class editor for C# on Linux.

written on 09 November 2013
